# Portfolio Overhaul Plan

## Overview
Transform the current portfolio into a killer GenAI/Agentic AI engineer's portfolio with a redesigned UI, cleaned dependencies, and comprehensive sections.

---

## Phase 1: Cleanup & Dependency Audit

### 1.1 Remove Unnecessary Packages
**Remove these packages (unused or redundant):**
- `@testing-library/dom` - No tests exist
- `@testing-library/react` - No tests exist
- `@vitejs/plugin-react` - Not needed for Next.js (Vite plugin doesn't integrate with Next.js)
- `jsdom` - Only used for vitest, which has no tests
- `vitest` - No test files exist
- `@radix-ui/react-separator` - Only used in `separator.tsx` which is never imported
- `@radix-ui/react-slot` - Only used in unused `badge.tsx` and `button.tsx` (shadcn defaults not used)
- `class-variance-authority` - Only used in unused shadcn components
- `tw-animate-css` - Only imported in globals.css but animations are custom-defined

**Keep:**
- `framer-motion` - Used in HeroSection and ReiatsuButton
- `lucide-react` - Used throughout sections
- `next-themes` - Used in ThemeProvider
- `@tanstack/react-query` - Keep for future data fetching (standard for portfolios with APIs)
- `clsx` + `tailwind-merge` - Used in `cn()` utility, used by GlassCard, HogyokuLink, ReiatsuButton

### 1.2 Remove Unused Files
- `components/Section.tsx` - Not imported anywhere
- `components/Skill.tsx` - Not imported anywhere
- `components/ui/badge.tsx` - Not imported anywhere
- `components/ui/button.tsx` - Not imported anywhere (using custom ReiatsuButton)
- `components/ui/card.tsx` - Not imported anywhere (using custom GlassCard)
- `components/ui/separator.tsx` - Not imported anywhere
- `vitest.config.ts` - No tests
- `vitest.setup.ts` - No tests
- `package-lock.json` - Conflicts with bun (AGENTS.md specifies Bun)

### 1.3 Clean package.json Scripts
- Remove `"test": "vitest"` script

## Phase 4: UI Redesign

### 4.1 Design System Improvements
- Keep the Aizen aesthetic (dark, purple/cyan accents)
- Enhance glassmorphism with better blur and borders
- Add consistent spacing scale
- Improve typography hierarchy
- Add micro-interactions on hover/focus

### 4.2 Animation Strategy
- Page load animations (staggered reveals)
- Scroll-triggered animations (Framer Motion `whileInView`)
- Hover effects on cards and buttons
- Smooth scroll behavior
- Loading states with skeleton screens

### 4.3 Responsive Design
- Mobile-first approach
- Breakpoints: sm (640px), md (768px), lg (1024px), xl (1280px)
- Collapsible navigation on mobile
- Stacked layouts on small screens

### 4.4 Accessibility
- Proper ARIA labels
- Keyboard navigation
- Focus states
- Color contrast compliance
- Reduced motion support

4. **Update existing components** - Enhance HeroSection, SkillsSection, ProjectsSection

## User Decisions

1. **Package Manager**: Remove `package-lock.json`, use Bun exclusively
2. **Testing**: Remove vitest, testing-library, jsdom entirely
3. **Content**: User will provide content later - use minimal placeholders
4. **Contact Form**: UI only with mailto/social links (no backend integration)
5. **Blog**: External links only (Medium, Dev.to, Hashnode)
6. **3D Elements**: Framer Motion only, no Three.js
